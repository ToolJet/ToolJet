# Workflow Automation

ToolJet Workflows is a visual backend automation builder for logic that runs server-side, independent of any app UI. Where app queries run in response to user interaction, workflows handle scheduled jobs, webhook processing, multi-step approval routing, and business logic that needs to run on a trigger — not on a click.

Workflows is available on the Business tier and above.

## How Workflows Differ from App Queries

App queries are tied to an app's lifecycle: they run when a user opens a page, clicks a button, or submits a form. The result goes back to the UI.

Workflows run server-side on their own trigger conditions, with no frontend required. A workflow can call multiple data sources in sequence, branch on conditions, loop over a dataset, and send results to external systems — all without a user present.

| | App Queries | Workflows |
|---|---|---|
| Trigger | User action / page load | Cron, webhook, manual, event |
| Execution | Client-initiated | Server-side |
| Frontend required | Yes | No |
| Multi-step branching | Limited | Full |
| Scheduled runs | No | Yes |
| Retry / error handling | Basic | Configurable |

## What Workflows Connect To

Workflows use the same data source connectors as apps. Any database, REST API, or SaaS tool available in the workspace can be called from a workflow node.

Common workflow actions:
- Query a database and write results to another system
- POST to a webhook or external API
- Send email or Slack notifications via SaaS connectors
- Read from a queue and process records in a loop
- Update a ToolJet Database table on a schedule

## Workflow Availability

Workflows is a paid feature, available on the Business tier and above. On Free and Basic tiers, app queries cover synchronous, user-triggered data operations. Evaluate Workflows if your use case requires server-side scheduled jobs, webhook processing, or multi-step automation independent of app sessions.

For current tier details, see the [ToolJet pricing page](https://www.tooljet.com/pricing).
