Security evaluation for an internal tools platform centers on three questions: What data does it touch? Who has independently verified its controls? What happens if there's an incident? This section covers ToolJet's security model, certifications, and the controls available for regulated environments.

## The Data Proxy Model

Understanding what data ToolJet touches is the foundation of any security assessment.

ToolJet is a **query router, not a data store**. When a user runs a query:

1. The browser sends the query request to the ToolJet server
2. ToolJet executes the query against your data source using stored credentials
3. The result is returned to the user's browser session
4. ToolJet does not log, cache, or persist query result data

**What ToolJet stores:** app definitions, user accounts, group assignments, data source connection configurations (credentials encrypted at rest), and audit logs. It does not store any data that flows through queries.

This model means: if your data source returns sensitive records, those records exist in the user's browser session but never in ToolJet's database.

## Audit Logs

Enterprise audit logs capture the full activity trail for a ToolJet workspace:

| Category | What's logged |
|---|---|
| Authentication | Login, logout, SSO events, failed attempts |
| App access | Which user accessed which app and when |
| Data operations | Query execution events (not result data) |
| Administration | User creation, deletion, group changes, permission updates |
| App changes | Version publishes, promotions, rollbacks |
| Data sources | Connection creation, modification, deletion |

Logs are available in the admin console and exportable via API for ingestion into Splunk, Datadog, Elastic, or any SIEM that accepts JSON.

## Self-Hosted Security Posture

For organizations that self-host, the security boundary is entirely within your infrastructure:

- **No outbound telemetry**: ToolJet can be configured to run with no outbound internet connections
- **Your key management**: encryption keys for credential storage are your responsibility to protect and rotate
- **Your network controls**: firewall rules, private subnets, and VPC peering for data source connections are configured at your infrastructure layer
- **Your audit pipeline**: logs are written to your Postgres database and accessible via API for forwarding to your SIEM

Air-gapped deployments support all enterprise features without requiring internet access at runtime. Images are mirrored to a private registry; no calls to external services are made after deployment.

## Vulnerability Disclosure

ToolJet maintains a responsible disclosure program. Security vulnerabilities can be reported through the ToolJet Security page. The team acknowledges reports within 48 hours and targets remediation within 90 days for validated vulnerabilities, with critical issues addressed sooner.
