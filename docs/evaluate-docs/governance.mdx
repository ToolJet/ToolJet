# Governance & Access Control

*This section covers how ToolJet controls who can do what: role-based access, custom group policies, workspace-level controls, and how IdP-driven group sync keeps permissions in sync with your directory.*

**Coming soon.** Content is being drafted for this section.

Topics this section will cover:

- Built-in roles — Admin, Developer, Viewer — and what each can do
- Custom group policies (Enterprise) — define arbitrary permission sets and apply them to user groups
- Workspace-level controls — isolating teams, apps, and data sources within a single ToolJet instance
- IdP group sync — automatic RBAC from Okta, Azure AD, Google Workspace, or any OIDC provider
- App-level permissions — per-app access control, public/private apps
- Component-level access rules — hide or disable UI elements based on user role
- Admin API — programmatic user and group management

In the meantime, see [User Management & Access Control](/docs/user-management/overview) for technical reference.
