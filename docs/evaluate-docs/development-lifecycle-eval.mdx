# Development Lifecycle

ToolJet apps are not deployed as code — but they do have versioning, environment promotion, and Git-backed backup. This section covers how apps move from development through staging to production, how teams collaborate on apps, and what CI/CD looks like for internal tools built on ToolJet.

## App Versioning

Every published change to a ToolJet app creates a version snapshot. Versions are immutable — once a version is published, its state is fixed.

## Multi-Environment Support

Apps can be promoted across environments (development → staging → production) within a single ToolJet instance. Each environment can have its own data source credentials, so the same app queries a staging database in staging and a production database in production — without modifying queries.

## GitSync

GitSync connects your ToolJet workspace to a Git repository. Apps are exported as JSON and committed to Git on each publish, creating a full version history that lives outside ToolJet.

What GitSync enables:

- **Backup**: every version of every app is in Git. If the ToolJet instance is lost, apps can be restored from the Git history
- **CI/CD**: trigger a deployment from a GitHub Actions or GitLab CI pipeline using the GitSync API — promote an app version to production as part of your standard release process
- **Audit trail**: Git history shows who changed what and when, independently of ToolJet's internal version list
- **Disaster recovery**: restore a workspace from a Git repository, including all app definitions and configurations

**GitSync is available on Business tier and above.**

## CI/CD Integration

The GitSync API allows external systems to interact with app promotion programmatically:

```
POST /api/v2/apps/{appId}/git/promote
```

Common pipeline patterns:
- On merge to main, promote the associated app from staging to production
- On PR open, auto-promote to a review environment for QA
- On tag, cut a production release and record the version in the pipeline run

The API uses the same authentication model as the rest of the ToolJet API — workspace API tokens scoped to the deployment environment.

## Backup & Restore

Apps can be exported individually or in bulk as JSON via the ToolJet API. The export includes component configuration, queries, event handlers, and variable definitions. Re-importing to a new instance restores the app to its exported state.

For full workspace backup (apps + users + groups + data source configs), GitSync provides the most comprehensive recovery path.
