# Development Lifecycle

*This section covers how ToolJet apps move from development to production: version control, multi-environment support, GitSync for CI/CD, and the upcoming branching feature for parallel development.*

**Coming soon.** Content is being drafted for this section.

Topics this section will cover:

- App versioning — immutable version snapshots, rollback to any prior version
- Multi-environment support — dev, staging, and production environments per app with promotion workflows
- GitSync — Git-backed backup and CI/CD integration. Apps exported as JSON, versioned in Git, deployable via API
- CI/CD with the GitSync API — trigger deployments from GitHub Actions, GitLab CI, or any pipeline
- Backup and recovery — export all apps and configurations; restore to a new instance
- App branching (upcoming) — parallel development branches for team collaboration without conflicts
- Release management — how app versions are promoted across environments

In the meantime, see [Development Lifecycle](/docs/development-lifecycle/overview) for technical reference.
